## Що це робить, простими словами

Коли Claude Code або OpenAI Codex CLI відкриває для вас файли, кожен їхній рядок потрапляє у вашу дорогу модель, за яку ви платите підпискою. Прочитати п'ять файлів — це вже близько 8 000 токенів (одиниць тексту, які лічить модель). Кілька таких операцій на день — і ви впираєтеся в тижневий ліміт.

`cheap-llm-router` додає одну невелику команду — **`cheap`**. Вона надсилає ваші файли значно дешевшій моделі (DeepSeek, Kimi, Gemini Flash — ваш вибір) і повертає коротке резюме, зазвичай близько 600 токенів. Ваша дорога модель отримує **відповідь**, а не сирий текст файлів.

Те саме для історії чату: коли ви ось-ось зробите «компакт» (стиснення розмови) у довгій сесії або хочете оновити документацію за тим, що тільки-но робили, команда `cheap extract` сама зробить резюме сесії, щоб дорога модель не перечитувала його повністю.

## Навіщо це може бути потрібно вам

- **Менше впиратися у тижневий ліміт.** Або не впиратися взагалі.
- **Той самий робочий процес.** Ви далі працюєте у Claude чи Codex; `cheap` підмикається через одне правило у файлі `CLAUDE.md` / `AGENTS.md`.
- **Будь-яка дешева модель.** Працює з усім, що сумісне з OpenAI API. За замовчуванням — `deepseek/deepseek-chat-v3-0324` (~$0.005 за типовий 4-файловий виклик, контекст 164K токенів) — перевірена нон-reasoning модель, що пройшла наш регресійний тест на 17 моделях. Див. [Як обрати модель](#як-обрати-модель) — повна таблиця.
- **Дві програми підтримуються.** Claude Code та OpenAI Codex CLI, з автоматичним розпізнаванням, у якій ви зараз сидите.
- **Чесний щодо обмежень.** Не допоможе у налагодженні, аудиті безпеки чи задачах, де треба точний текст — у таких випадках він каже дорогій моделі читати файли самій.

### Що це означає для тижневого ліміту підписки

## Швидкий старт (3 кроки)

### 1. Встановити

```bash
pipx install git+https://github.com/Lexus2016/cheap-llm-router.git
```

(Беремо `pipx`, а не `pip`, щоб у програми було своє ізольоване Python-середовище. На macOS: `brew install pipx`.)

### 2. Задати ключ OpenRouter

Зареєструйтеся безкоштовно на <https://openrouter.ai/>, поповніть кілька доларів, скопіюйте ваш ключ, потім:

```bash
echo 'export OPENROUTER_API_KEY=sk-or-v1-...' >> ~/.zshrc
source ~/.zshrc
cheap config check       # → OK
```

### 3. Розповісти Claude / Codex про новий інструмент

```bash
cheap install-claude-rule
```

Готово. Відтепер, коли Claude чи Codex збирається прочитати кілька файлів просто щоб зрозуміти контекст, вона автоматично використає `cheap`.

### Оновлення правила пізніше

Якщо ви повторно запустите `cheap install-claude-rule` після нового релізу, команда залишиться тихою, коли локальне правило вже актуальне. Якщо вийшла новіша версія правила — побачите підказку:

```
already installed at /Users/you/.claude/CLAUDE.md (rule v1; v2 available
  — run with --force to upgrade, this REPLACES the block including any
  local edits)
```

`--force` перезаписує весь блок `## Cheap LLM …`, тож будь-які ваші локальні правки в ньому (наприклад, власні записи в «Past mistakes») будуть втрачені — скопіюйте їх окремо заздалегідь, якщо хочете зберегти.

### Опціонально: PreToolUse hook (тільки Claude Code)

Правило само по собі покладається на те, що агент сам згадає про делегацію перед кожним `Read`. На довгих сесіях ця дисципліна падає нижче 5%. Опціональний hook закриває цей розрив тим, що **в момент** виклику `Read` додає підказку прямо в контекст агента — але тільки коли читання дійсно delegate-worthy (повний файл ≥200 рядків або ≥2 повних читання за кілька попередніх turn'ів). Інші Reads (короткі файли, line-targeted, після Edit на цей же файл, secrets, зображення) проходять без шуму.

```bash
cheap install-hook        # додає PreToolUse:Read запис у ~/.claude/settings.json
```

Hook ніколи не блокує: завжди повертає `permissionDecision: "allow"` і додає лише `additionalContext` як підказку. Інші агенти (Codex CLI, Cursor, Aider, Cline, Continue, OpenCode, Gemini CLI) поки що не мають такого ж hook-API — для них єдиним механізмом делегації лишається правило з `cheap install-rule`.

## For AI agents (auto-install)

Якщо ти AI-агент (Claude Code, Codex CLI, Gemini, Cursor тощо), що читає цей README щоб встановити `cheap` для свого користувача — виконай ці кроки по порядку. Зупинись і запитай користувача, якщо для якогось кроку не маєш потрібної інформації.

```bash
# 1. Переконатися що pipx встановлений (приклад для macOS):
command -v pipx >/dev/null 2>&1 || brew install pipx
pipx ensurepath

# 2. Встановити (або примусово оновити) cheap-llm-router:
pipx install --force git+https://github.com/Lexus2016/cheap-llm-router.git

# 3. Перевірити статус — повідомить про відсутність OPENROUTER_API_KEY:
cheap config check
```

Якщо `cheap config check` повідомляє про відсутній `OPENROUTER_API_KEY`:

1. **ЗУПИНИСЬ і запитай користувача** його ключ OpenRouter API — він створює його на <https://openrouter.ai/keys>.
2. Додай у профіль shell (рекомендовано — секрет не потрапляє у файл під git):
   ```bash
   echo 'export OPENROUTER_API_KEY=sk-or-v1-...' >> ~/.zshrc
   source ~/.zshrc
   ```
   Для ізольованих оточень — натомість редагуй `~/.config/cheap-llm/config.yaml`, закоментуй рядок `# api_key_env:` і встав `provider.api_key: "sk-or-v1-..."`.

Потім продовжуй:

```bash
# 4. Встановити правило у ~/.claude/CLAUDE.md і ~/.codex/AGENTS.md:
cheap install-claude-rule

# 5. Smoke-test — має надрукувати markdown-резюме, НЕ порожній/короткий вивід:
cheap read README.md -q "що це за проєкт?"
```

**Перевір перш ніж сказати користувачу «готово»:**

- `cheap config check` повертає `OK`.
- `cheap config show` показує `model: deepseek/deepseek-chat-v3-0324` (або іншу перевірену non-reasoning модель — див. [Як обрати модель](#як-обрати-модель)).
- Smoke-test повертає непорожній контент (>500 байтів на stdout).

**Якщо smoke-test повертає порожній / дуже короткий вивід**, налаштована модель є reasoning-здатною (`deepseek/deepseek-v4-pro`, `kimi-k2.6`, `kimi-k2-thinking`, `glm-5.1`, `glm-4.7-flash`, `mimo-v2.5`) — вона витратила весь cap `max_summary_tokens` на приховані thinking-токени, не залишивши видимого виводу. Зміни рядок `model:` у `~/.config/cheap-llm/config.yaml` на перевірену non-reasoning модель і повтори smoke-test.

**Перш ніж брати нову модель за дефолт**, запусти регресійний скрипт що ловить усі три відомі режими провалу (порожній вивід, зламаний cap, втрата файлів):

```bash
git clone https://github.com/Lexus2016/cheap-llm-router.git /tmp/clr
/tmp/clr/scripts/test-model.sh <provider/model-id>
```

Скрипт автоматично робить бекап конфігу, виконує single-file + 4-file тести, друкує вердикт PASS / NOT RECOMMENDED з причинами, і відновлює оригінальний конфіг при виході (навіть на Ctrl-C).

## Дві основні команди

### `cheap read FILE…`

Робить резюме списку файлів. Використовуйте, коли у вас є **питання** про код, а не задача змінити його.

```bash
cheap read src/auth.py src/db.py src/api.py -q "як тут зроблений вхід?"
```

Ви отримаєте резюме на ~600 токенів у markdown-форматі, плюс один рядок телеметрії (звіт про виклик), де чітко написано, скільки воно коштувало:

```
[cheap] files=3 input_chars=15823 output_tokens=587 model=deepseek/deepseek-chat-v3-0324 elapsed_ms=2143
```

### `cheap extract`

Робить резюме **історії поточної сесії чату**. Найкорисніше прямо перед `/compact` у Claude або коли ви хочете оновити документацію за тим, що тільки-но робили.

```bash
cheap extract -q "що ми сьогодні вирішили?"
```

Програма сама розуміє, чи ви у Claude Code, чи у Codex CLI, і знаходить правильний файл сесії. Передайте `--mode messages-only`, щоб залишити лише повідомлення (без шуму від службових викликів інструментів), або `--tail 50`, щоб залишити лише 50 останніх повідомлень.

## Де лежить API-ключ

Два підтримувані варіанти. Виберіть один.

| Де лежить ключ | Коли використовувати |
|---|---|
| **Змінна середовища** `OPENROUTER_API_KEY` | **Рекомендовано.** Ключ не потрапляє у жоден файл. |
| **Поле `api_key:`** у `~/.config/cheap-llm/config.yaml` | Ізольовані оболонки, GUI-запускачі, де змінні середовища незручні. |

Якщо задані обидва, виграє той, що у YAML-файлі (явно вказане має пріоритет).

Коли ключ лежить у YAML, `cheap config show` автоматично маскує його як `***REDACTED***`. Але `cat $(cheap config path)` цього не робить — тому тримайте файл подалі від git, синхронізації iCloud / Dropbox і скріншотів.

## Як обрати модель

Не кожна дешева модель підходить. Три режими провалу яких треба уникати:

1. **Reasoning-моделі** (наприклад `deepseek/deepseek-v4-pro`, `kimi-k2.6`, `kimi-k2-thinking`, `glm-5.1`, `glm-4.7-flash`, `mimo-v2.5`) — мовчки витрачають весь cap `max_summary_tokens` на приховані thinking-токени, повертаючи порожній видимий output.
2. **Моделі що ігнорують cap** (наприклад `kimi-k2.5`, `z-ai/glm-4.7`) — перевищують cap у 2-8 разів, з'їдаючи ваш token-бюджет.
3. **Моделі що втрачають покриття** (наприклад `kimi-k2`, `gpt-5.4-nano`) — фокусуються на перших 1-2 файлах і ігнорують решту при multi-file читаннях.

### Перевірені моделі (4-файловий тест 79K chars input, cap=1500 токенів, sweep 2026-05-05)

| Модель | Час | Tokens | Покриття | Context | $/4-file | Нотатки |
|---|---|---|---|---|---|---|
| **`deepseek/deepseek-chat-v3-0324`** ★ | **36с** | 1237 | **4/4** | 164K | $0.005 | DEFAULT — стабільна, швидка |
| `google/gemma-4-31b-it` | 50с | 1500 | 4/4 | 262K | $0.003 | ~35% дешевша, трохи повільніша |

\* `deepseek/deepseek-v4-flash` — використовуй лише коли потрібен її 1M контекст. Поєднуй з `max_input_chars: 2500000` у `~/.config/cheap-llm/config.yaml`.

## Часті питання

**Чи знатиме сама модель, коли використовувати `cheap`, а коли ні?**
Так — `cheap install-claude-rule` записує точний чек-ліст у `CLAUDE.md` (і `AGENTS.md`). Правило каже: *використовуй `cheap` тільки коли потрібна **відповідь**, а не точний текст. Не використовуй для редагування, аудиту безпеки чи всього, де треба цитувати чи порівнювати рядки точно*.

**Чи безпечно це з секретами?**
`cheap read` за замовчуванням відмовляється надсилати файли, чиї імена відповідають шаблонам `.env*`, `*.key`, `*.pem`, `id_rsa`, `credentials.json` тощо. Можна перевизначити через `--include-sensitive` (тоді буде попередження), але краще цього не робити.

**Куди реально йдуть мої дані?**
В OpenRouter, а звідти — до того провайдера, який ви обрали (DeepSeek, Moonshot, Google…). Якщо ви вже користуєтеся Claude чи Codex, ваш код уже проходить через одного стороннього AI-провайдера. Цей інструмент додає ще одного, але якісно нічого не змінює.

**Скільки реально економиться?**
8× дешевше ніж згодувати файли в Claude Opus 4.7 напряму, 6× проти Sonnet 4.6, ~3× проти GPT-5-codex. ~$1.65 на одного активного розробника на день на цінах Opus 4.7, ~$45/міс. Див. [Реальні цифри](#реальні-цифри--наскільки-це-здешевшує-роботу) вище — повна таблиця. `cheap` пише один рядок телеметрії на кожен виклик, тож можна перевірити математику на власному робочому процесі.

**Я не користуюся Claude. Можу взяти це з будь-якою оболонкою?**
Так — `cheap read` і `cheap extract` — це звичайні shell-команди. Запускайте вручну, перенаправляйте результати, робіть що хочете. Крок `install-claude-rule` — необов'язковий.
